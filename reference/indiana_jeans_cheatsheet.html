Indiana Jeans Project Cheatsheet

1. JSON Server Naming & Expand Rules

  • Database (in db.json) arrays/tables must be **lowercase plurals**:
    submissions, socioLocations, etc.
  • Foreign key fields must be named as singular + Id, referencing the parent table.
    E.g. in submissions, have a field socioLocationId pointing to a socioLocation.
  • Expand usage syntax: ?_expand=singularTableName, where singularTableName matches the thing you‚Äôre expanding (without the ‚ÄúId‚Äù).
    Example: GET /submissions?_expand=socioLocation

2. fetch(), GET, POST, Async

  • Use fetch(url) to make requests.
  • For GETs, you can append query parameters like ?_expand=‚Ä¶.
  • POST pattern:

  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(dataObject)
};
await fetch("http://localhost:8088/submissions", postOptions);
  • Use async / await for cleaner, linear‚Äëstyle asynchronous code.
    Always await response.json() when you want the parsed JSON results.

3. Transient State & Input Parsing

  • transientState: holds temporary user choices before submitting:
    ```js const transientState = { ownsBlueJeans: false, socioLocationId: 0 } ```
  • Use setter functions to update it: 
    
export const setOwnsBlueJeans = (chosenOwnership) => {
  transientState.ownsBlueJeans = chosenOwnership;
};
export const setSocioLocationId = (chosenLocation) => {
  transientState.socioLocationId = chosenLocation;
};
  • Parse values from form inputs so they‚Äôre the right datatype: 
    
export const parseBoolean = (value) => {
  return value === "true" ? true : (value === "false" ? false : null);
};

export const parseInteger = (value) => {
  const parsed = parseInt(value, 10);
  return isNaN(parsed) ? null : parsed;
};

4. Form Validation Before Submission

  • Check both fields are valid:
    • ownsBlueJeans must be a boolean (true or false).
    • socioLocationId must be a number > 0.
  • If invalid, show a specific alert, then return; so you don‚Äôt do a POST.
  • Example: 
    
if (!ownsValid && !locationValid) {
  alert("Please answer both questions before submitting.");
  return;
} else if (!ownsValid) {
  alert("Please answer the question about owning jeans.");
  return;
} else if (!locationValid) {
  alert("Please select your location type.");
  return;
}
// Both valid here -> proceed with fetch POST

5. Event Handling & Rendering Architecture

  • Modular event handlers: functions like handleOwnershipChange, handleLocationChoice live in their own modules so they can be imported centrally.
  • Central Event Hub: one place (e.g. eventHub.js) where you add event listeners: 
    
document.addEventListener("change", (event) => {
  if (event.target.name === "ownsJeans") {
    handleOwnershipChange(event);
  }
  if (event.target.name === "location") {
    handleLocationChoice(event);
  }
});
  • Rendering functions: each component returns HTML; use await when needed (for fetching data). Then assemble them in a master render() function.
  • Import eventHub as module in main: ensures listeners are registered before user interacts with form: 
    
import "./eventHub.js";
import { JeanChoices } from "./JeanChoices.js";
import { LocationChoices } from "./LocationChoices.js";
// etc.

6. Example Render Flow

  1. Load HTML page. index.html includes main.js.
  2. Main.js: imports eventHub.js; calls render().
  3. render(): calls component functions:
    JeanChoices(), await LocationChoices(), SubmissionButton(), await SubmissionList(), then sets container.innerHTML.
  4. User interacts: changes radio ‚Üí global event listener in eventHub.js sees it, calls correct handler. Handlers update transientState.
  5. User attempts submission: saveSurveySubmission() checks transient state; alerts if incomplete; otherwise, POSTs to server.
